# ShuffleGolf — Sistema de Idiomas (i18n)

Soporte de **Castellano (es)** e **Inglés (en)** con selector en el menú principal y en una
ventana flotante de ajustes (engranaje) accesible durante la partida.

## Esquema de carpetas y archivos

```
ShuffleGolf-main/
├── index.html              # Marcado con atributos data-i18n / data-i18n-html / data-i18n-title
│                           # + botón engranaje (#gear-btn) y modal #settings-overlay
├── css/
│   └── style.css           # Estilos de #gear-btn, .settings-panel, .settings-row, .lang-switch
├── src/
│   ├── i18n.js             # ★ NUEVO — motor de traducción (diccionarios es/en + API)
│   ├── audio.js            # (sin cambios)
│   ├── golfers.js          # + diffKey / descKey por golfista (claves de traducción)
│   ├── data.js             # M_TYPES (misiones) usan t(); gemas se traducen por id
│   ├── drawing.js          # (sin cambios relevantes)
│   ├── cards.js            # Deck builder, tienda, slot, recompensas → t()
│   └── core.js             # HUD, scorecard, penalizaciones, fin de hoyo → t()
├── legacy/
│   └── script.js           # código muerto (no tocar)
└── docs/
    └── I18N.md             # este documento
```

**Orden de carga de scripts** (en `index.html`): `i18n.js` se carga **el primero** para que la
función global `t()` esté disponible cuando el resto de módulos la usen.

## API de i18n.js

| Función | Descripción |
|---------|-------------|
| `t(key, params)` | Devuelve el texto traducido. `params` interpola `{nombre}`. Ej: `t('m_drive', {v: 240})` |
| `setLang('en'|'es')` | Cambia el idioma, lo persiste en `localStorage` y refresca la UI |
| `getLang()` | Devuelve el idioma actual |
| `itemName(item)` | Nombre localizado de un objeto (las gemas se traducen por `id`; el resto mantiene su nombre) |
| `applyStatic(root)` | Rellena los elementos marcados con atributos `data-i18n*` |
| `I18N` | Objeto con los diccionarios `{ es: {...}, en: {...} }` |

### Idioma por defecto
Primer arranque: se detecta el idioma del navegador (`en-*` → inglés, resto → castellano).
Una vez el usuario elige idioma, la preferencia se guarda en `localStorage['dg_lang']`.

## Marcado estático en HTML

```html
<span data-i18n="hud_hole"></span>          <!-- textContent -->
<div  data-i18n-html="menu_warning"></div>  <!-- innerHTML (permite <br>) -->
<button data-i18n-title="settings_open">    <!-- atributo title -->
<button data-lang="es">ES</button>          <!-- botón de cambio de idioma -->
```

## Cómo añadir una cadena nueva

1. Añade la clave a **ambos** diccionarios (`es` y `en`) en `src/i18n.js`.
2. En HTML estático: usa `data-i18n="tu_clave"`.
3. En JS dinámico: usa `t('tu_clave')` (o `t('tu_clave', {param: valor})`).

## Cómo añadir un tercer idioma

1. Añade un bloque nuevo a `I18N` (p. ej. `fr: { ... }`) con todas las claves.
2. Añade un botón `<button data-lang="fr">FR</button>` en el menú y en el modal de ajustes.
No hace falta tocar nada más: los botones `data-lang` se conectan automáticamente.

## Notas de implementación

- **Sombreado de `t`**: en `core.js` las funciones `endShot`, `finishTurn` y `applyPenalty`
  tienen una variable local `t` (el terreno). Dentro de ellas se usa `window.t(...)` para invocar
  la traducción y evitar el conflicto de nombres.
- **Datos vs. presentación**: los golfistas conservan `difficulty`/`desc` originales como datos,
  pero la UI usa `diffKey`/`descKey` para traducir. Las dificultades mantienen sus clases CSS
  (`diff-Facil`, etc.) vía `diffKey`.
- **Términos universales de golf** (Driver, 3 Wood, Power, Aero, Birdie, Par, Eagle…) se mantienen
  iguales en ambos idiomas a propósito.
